# 课题：《网络病毒的防范》

## 一、 教学设计 (Lesson Design V6)

### 零、 教学内容切片声明（AI 诊断）
- **逻辑主线与聚焦**：网络病毒传播途径的演变 → 针对新途径的多层防毒体系建设。重点突破“多层防御体系的架构（桌面-服务器-网关/防火墙）”。
- **轻量串联与延后点**：轻量串联病毒通过邮件和下载传播的高效性；删减抽象的“三大特点（层次、集成、自动化）”，将其留作选做作业，保证课堂实操感。

### 一、 导入新课（约 1 分钟，约 120 字）
- **调用的策略**: [生活情境导入 INT-001]
- **内容设计**: 抛出“勒索病毒”瘫痪校园网的真实案例。
- **预期师生互动**: 提问为何病毒传播速度远超从前 → [个人思考] → 学生回答 → 教师请坐并具体评价 → 板书课题。

### 二、 新课讲授（约 5 分钟，约 600 字）
#### 重点知识 1：病毒传播新途径
- **调用的策略**: [情境代入提问 QUE-003]
- **教学逻辑链**: 
  - **观察/阅读**：结合教材思考网络带来的新途径。
  - **提问与互动**：[同桌讨论] 你们平时最容易从哪里中招？
  - **学生回答处理**：请坐 → 复述提炼（邮件、下载、共享） → 评价：结合生活，概括完整。
  - **建构与板书**：[随讲随写板书：1. 传播途径：邮件/下载/共享]
- **阶段性小结**：单机防杀已不够，需构建体系化防御。

#### 重点知识 2：多层防御的网络防毒体系
- **调用的策略**: [认知冲突提问 QUE-002] + [拓扑与业务实体映射策略 EXP-003]
- **教学逻辑链**: 
  - **观察/阅读**：展示仅有终端防护的企业网络拓扑图。
  - **提问与互动**：[小组讨论] 黑客发来带毒邮件，这套防御会从哪里崩溃？该在图上哪三个关键点设防？
  - **学生回答处理**：请坐 → 复述提炼（终端、服务器、边界） → 评价：分析准确，逻辑严密。
  - **建构与板书**：[随讲随写板书：2. 多层防御：桌面→服务器→网关/防火墙]
- **阶段性小结**：源头、枢纽和边界三管齐下，才能构建立体防御。

### 三、 巩固练习（约 1.5 分钟，约 180 字）
- **调用的策略**: [网络拓扑排错互动 INT-002]
- **综合练习设计**: 展示错误防毒拓扑（网关“裸奔”，邮件过滤装在桌面）。
- **练习反馈**: 巡视，引导学生纠错。

### 四、 课堂小结（约 1 分钟，约 120 字）
- **调用的策略**: [归纳总结提问 QUE-007]
- **小结方式**: 预设请学生总结三道防线 → 教师补充提升 → 呼应目标。
- **思政融入点**: 国家网络安全长城，科技报国。

### 五、 布置作业（约 0.5 分钟，约 60 字）
- **必做作业**: 画出三层防毒网络拓扑图。
- **选做作业**: 思考高级防毒体系的“自动化与集成性”在真实企业中如何落地。

## 二、 考场逐字稿 (Teaching Script V7 极净/客体化/建构版)

**【一、 导入新课】**
同学们，上课！大家看大屏幕上这张新闻截图。几年前，“勒索病毒”在一夜之间席卷全球，许多大学的实验室、医院的系统瞬间瘫痪，文件被全部加密锁死。大家思考一下，在以前用软盘拷贝文件的年代，病毒传播得慢吞吞的，为什么现在却能一夜之间爆发？第一排的男同学，请你来说说看。
好，请坐。这位同学提炼出了关键信息，他认为是“网络互联”加快了速度。
正如他所说，互联网的高速公路不仅传输数据，也给病毒插上了翅膀。面对这种立体打击，我们该如何防守？今天我们就来学习网络安全的核心一课。
**[板书：网络病毒的防范]**

**【二、 新课讲授】**
既然要防，得先知道病毒从哪来。大家快速浏览课文第一段，同桌之间互相讨论一下，除了传统的 U 盘，现在病毒最喜欢藏在哪些通道里攻击我们？
大家讨论得很热烈。第一组的这位代表，你来总结一下你们的发现。
请坐。第一组结合生活经验，概括得很完整！他们提到了电子邮件、来历不明的文件下载，还有局域网内的共享文档。
**[板书：1. 传播途径：邮件 / 下载 / 共享]**
同学们，刚才大家找出的这些途径证明了一点：在企业网络里，如果只给电脑装个杀毒软件，只要网络里有一台机器染毒，整个系统就会瘫痪。
面对这种情况该怎么办？请看大屏幕上的这张企业网络拓扑图。如果现在有一封带毒的邮件从外网发进来，大家分小组讨论一下，为了保护内网安全，我们应该在这张图上的哪几个关键位置，设置防毒的“关卡”？给大家两分钟时间。
好，时间到。第三小组的组长，请你来分享一下你们的防御方案。
请坐。他们组分析得非常准确，逻辑很严密。他们抓住了网络数据流向的咽喉。
正如这组同学所梳理的，第一道关卡，是最边缘的个人电脑，因为它是最终感染源，所以要在桌面系统设防；第二道关卡，是内部通信的枢纽，也就是群件和邮件服务器；而最重要的一道防线在哪里？对，大家一起说，在内外网交界的网关和防火墙！在这里安装扫描器，把敌对代码挡在门外。
这种从里到外、层层设卡的理念，在安全领域就叫作“多层防御体系”。
**[板书：2. 多层防御：桌面 → 服务器 → 网关/防火墙]**

**【三、 巩固练习】**
为了检验大家是不是真懂了，我们来看大屏幕上一道实战诊断题。这里有一张某公司的网络部署图，管理员把过滤网页和恶意邮件的模块，全装在了员工个人的电脑上，网关处却什么也没装。大家在草稿纸上推演一下，这会造成什么后果？
刚才我在巡视时，发现这位戴眼镜的同学眉头紧锁。请你来说说你的顾虑。
请坐。这位同学找出了破绽。他敏锐地指出，等恶意数据到了个人电脑再拦截，已经消耗了内网带宽，为时已晚。必须把这种全局扫描的关卡前置到网关。大家把图上的错误配置在草稿纸上纠正过来。

**【四、 课堂小结】**
课程接近尾声，今天我们建立了一个立体的防御观。谁能用自己的话，把我们这三道防线再梳理一遍？班长，你来。
概括得很完整，请坐。班长准确复述了桌面、服务器和网关的多层防护体系。
同学们，网络世界没有绝对的安全。我们今天讲的是企业的防毒拓扑，放大来看，我们国家的网络安全长城，同样需要这样严密的多层防线。希望大家不仅学好技术，未来更能做祖国信息安全的忠诚卫士！

**【五、 布置作业】**
布置今天的分层作业。
必做题：请大家独立画出包含三层防线在内的安全网络拓扑图。
选做题：教材最后提到了高级防毒体系具有“自动化”和“集成性”，请大家课后查阅资料，思考这在真实企业中是如何落地的。
今天的课就上到这里，同学们，下课！
