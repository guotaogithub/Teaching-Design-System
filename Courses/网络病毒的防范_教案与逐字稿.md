# 课题：《网络病毒的防范》

## 一、 教学设计 (Lesson Design V6)

### 零、 教学内容切片声明（AI 诊断）
- **逻辑主线与聚焦**：网络病毒传播途径的演变 → 针对新途径的多层防毒体系建设。重点突破“多层防御体系的架构（由外到内：网关/防火墙-服务器-桌面）”。
- **轻量串联与延后点**：轻量串联病毒通过邮件和下载传播的高效性；删减抽象的“三大特点（层次、集成、自动化）”，将其留作选做作业，保证课堂实操感。

#### 重点知识 2：多层防御的网络防毒体系 (由外到内的工程逻辑)
- **调用的策略**: [认知冲突提问 QUE-002] + [拓扑与业务实体映射策略 EXP-003]

## 二、 考场逐字稿 (Teaching Script V8 极净/客体化/工程逻辑版)

**【一、 导入新课】**
同学们，上课！大家看大屏幕上这张新闻截图。几年前，“勒索病毒”一夜之间席卷全球，许多大学的实验室、医院的系统瞬间瘫痪，文件被全部加密锁死。大家思考一下，在以前用软盘拷贝文件的年代，病毒传播得慢吞吞的，为什么现在却能爆发得这么快？第一排的男同学，请你来说说看。
好，请坐。这位同学提炼出了关键信息，他认为是“网络互联”加快了速度。
正如他所说，互联网不仅传输正常数据，也给病毒插上了翅膀。面对这种立体打击，我们该如何防守？今天我们就来学习网络安全的核心一课。
**[板书：网络病毒的防范]**

**【二、 新课讲授】**
既然要防，得先知道病毒从哪来。大家快速浏览课文第一段，同桌互相讨论一下，除了传统的 U 盘，现在病毒最喜欢藏在哪些通道里攻击我们？
大家讨论得很热烈。第一组的这位代表，你来总结一下你们的发现。
请坐。第一组结合生活经验，概括得很完整！他们提到了电子邮件、来历不明的文件下载，还有局域网内的共享文档。
**[板书：1. 传播途径：邮件 / 下载 / 共享]**
同学们，刚才大家找出的这些途径证明了一点：在企业网络里，由于病毒可以通过网络满天飞，如果你只给自己的电脑装个杀毒软件，是防不住的。
那面对外部的疯狂攻击，企业网络到底该怎么设防？请看大屏幕上的拓扑图。假设黑客从外部 Internet 发起了一次病毒攻击。大家分小组讨论一下，为了把病毒隔离在尽可能远的地方，顺着外部数据流向内部的路径，我们应该“由外到内”在哪三个关键位置，依次设立“防毒关卡”？给大家两分钟。
好，时间到。第三小组的组长，请你来分享一下你们的防御方案。
请坐。他们组分析得非常准确，完全符合网络工程的防守逻辑。
正如这组同学所梳理的，面对外部攻击，我们布置的第一道、也是最重要的一道防线，必须设立在内外网的交界处。大家一起说，这叫什么设备？对，网关和防火墙！在这里安装扫描器，把敌对代码直接挡在大门外。
如果病毒狡猾，突破了大门进入了内部，那我们的第二道关卡，就是内部通信的枢纽，也就是群件和邮件服务器，在这里拦截交叉感染。
最后，如果病毒还是漏网了，我们的最后一道底线才是哪儿？没错，是最末端的用户桌面系统。
这种由外到内、层层拦截的理念，在安全领域就叫作“多层防御体系”。
**[板书：2. 多层防御：网关/防火墙 → 服务器 → 桌面]**

**【三、 巩固练习】**
为了检验大家是不是真懂了，我们来看大屏幕上一道实战诊断题。这里有一张某公司的防毒部署图，网络管理员把专门过滤有害网页和邮件的防毒墙，全装在了员工个人的电脑端，却让最外层的网关处于“裸奔”状态。大家在草稿纸上推演一下，这会造成什么后果？
刚才我在巡视时，发现这位戴眼镜的同学眉头紧锁。请你来说说你的顾虑。
请坐。这位同学找出了破绽。他敏锐地指出，等恶意数据流到了最内部的个人电脑再拦截，它早就已经占用了大量的内网带宽，甚至已经感染了服务器，这完全是本末倒置！大家把图上的错误配置在草稿纸上纠正过来，把防线推到大门口去。

**【四、 课堂小结】**
课程接近尾声，今天我们建立了一个由外向内的立体防御观。谁能用自己的话，把我们这三道防线再梳理一遍？班长，你来。
概括得很完整，请坐。班长准确复述了网关、服务器和桌面的多层防护体系。
同学们，今天我们讲的是企业的防毒拓扑。放大来看，我们国家的网络安全长城，同样需要把防线筑在最前沿。希望大家不仅学好安全技术，未来更能做祖国信息安全的第一道防火墙！

**【五、 布置作业】**
布置今天的分层作业。
必做题：请大家独立画出包含由外到内三层防线在内的安全网络拓扑图。
选做题：教材最后提到了高级防毒体系具有“自动化”和“集成性”，请大家课后查阅资料，思考这在真实企业中是如何落地的。
今天的课就上到这里，同学们，下课！
