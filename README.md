# Teaching Design System (计算机网络试讲教学设计系统)

中职计算机网络教师招聘 10 分钟无生试讲 Codex Skill。根据教材内容自动生成教案骨架和 950-1100 字考场逐字稿，覆盖内容选取、教学设计、逐字稿扩写、审查和修订的完整闭环。

## 设计理念

一切围绕考场实战。不为流程而流程——所有设计、互动、评价和板书都服务于知识建构和教学目标达成。Skill 内核采用渐进披露架构：根据任务类型只加载必要规则，避免上下文膨胀。

`Prompts/逐字稿生成标准V2.0.md` 为原始母版，保留用于追溯和审计；日常执行规则已拆入 `references/`。

## Skill 功能路由

Skill 支持 5 种任务模式，根据用户请求自动选择最小必要流程：

1. **创建新课程**：提供课题或教材片段 → 内容选取 → 教学设计骨架 → 逐字稿 → 自检修订
2. **只生成教学设计**：提供材料 → 五环节教学设计骨架 + 内容取舍说明
3. **只生成逐字稿**：提供已有教案 → 扩写 950-1100 字逐字稿 + 字数统计 + Checklist
4. **Review 现成逐字稿**：提供逐字稿 → 按标准诊断 → 分级问题 + 修改建议
5. **修订现成逐字稿**：提供逐字稿 → 诊断 → 修正 → 最终稿 + 修改说明

## 目录结构

- `SKILL.md`：Skill 入口，含渐进披露规则和策略文件映射
- `references/`：Skill 执行内核，6 个文件按需加载
  - `standard-core.md` — 基础总规（时间、容量、结构、互动、语言）
  - `content-selection.md` — 内容选取与轻量串联/合理留白策略
  - `lesson-design.md` — 教学设计骨架生成与策略选择
  - `script-writing.md` — 逐字稿扩写规则与互动写法
  - `script-review.md` — 逐字稿审查清单与严重度分级
  - `script-revision.md` — 逐字稿修订工作流与常见修改策略
- `Strategy-Library/`：9 个结构化策略文件，覆盖导入、提问、互动、讲解、过渡、板书、练习、评价、收束
- `Prompts/`：仅保留 V2.0 原始母版，用于追溯对照
- `Courses/`：优质课例存档

## 使用方式

在 Codex 中直接描述需求即可，Skill 自动路由到对应工作流：

- **生成新课程**：提供课题名称和教材内容片段
- **只出教案**：指定只需要教学设计骨架
- **审稿**：粘贴已有逐字稿请求 review
- **改稿**：粘贴逐字稿请求修改或优化

所有输出附带纯讲话字数统计和基于标准的 Checklist 自检报告。
