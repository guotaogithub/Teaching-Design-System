# Teaching Design System (计算机网络试讲教学设计系统)

本项目致力于为**计算机网络教师招聘面试（10分钟无生试讲）**提供一套标准化、可复用、并可由大语言模型（AI）自动化生成高质量教案与逐字稿的解决方案。

## 🎯 核心理念 (Core Philosophy)
拒绝大而全的传统教育学理论，一切为了**考场拿分**与**落地实战**。
坚持**“策略模块化 → 教学设计骨架化 → 逐字稿标准化”**的三步走工作流，用 AI 降低撰写试讲稿的门槛，同时利用严格的标准约束消除 AI 生成的“假大空”问题。

## 🚀 北极星工作流 (North Star Workflow)
1. **Strategy Library (教学策略库)**：沉淀适用于 10 分钟试讲的高频、高分互动与讲解策略（如“网络故障导入”、“命令行推演互动”、“抓包逆向推导”等），作为教案生成的结构化积木。
2. **Lesson Design Engine (教学设计生成引擎)**：利用专用的 Prompt 引擎（内置“大章节切片剪枝”逻辑），指引 AI 从策略库提取积木，精准搭建 5 步教学设计（装配图）。
3. **Teaching Script (逐字稿生成)**：基于教学设计骨架，严格按照字数限制（950-1100字）、高分互动循环（学生回答→请坐→复述提炼→自然推进）以及思政融入等要求，生成最终的考场背诵稿。

## 📂 目录结构 (Directory Structure)
- `Standards/`：项目的总体宪法与顶层标准（明确目标、原则、规范）。
- `Strategy-Library/`：**核心资产区！** 包含导入、提问、互动、讲解、过渡、板书、练习、评价、总结等 9 大模块的结构化策略卡片。
- `Lesson-Design-Engine/`：驱动大模型生成教学设计教案的核心引擎（包含内容取舍切片策略及教案生成 Prompt V4）。
- `Prompts/`：用于生成最终逐字稿的规则、标准与核心提示词。
- `Courses/`：用于存放未来使用引擎生成的优质教案和逐字稿实例（Course Library）。

## 🛠 如何使用 (How to use)
1. **输入课题**：准备好你的试讲题本（教材内容片段）。
2. **生成教案骨架**：将 `Lesson-Design-Engine/Lesson_Design_Prompt_V4.md` 的内容与 `Strategy-Library/` 下的策略文本投喂给大模型，让大模型自动切片并生成标准化的教学设计。
3. **生成逐字稿**：结合生成的教学设计，利用 `Prompts/逐字稿生成标准V2.0.md`，让大模型输出最终完美控制在 10 分钟左右的逐字稿。
